Giao diện Frontend: hướng tới user - Xác định chủ đề project sẽ làm để xác định trước các trang có thể có: + Bán hàng (laptop, quần áo, bia, sách) + Thi online: admin tạo ra các đề thi, user đăng ký để thi + chia sẻ file: admin tạo 1 nơi lưu tập trung các file, user có thể trả phí/free để tải file + Tin tức: ko nên làm trang tin tức (ko nên làm) - Sau khi xác định chủ đề, cần phải liệt kê các trang có thể có để xây dựng giao diện VD: với trang bán hàng: trang chủ, giỏ hàng, danh mục sp, chi tiết sp, thanh toán, login/register, user info, liên hệ, ... - Xây dưng giao diện như thế nào sau khi xác định đc các trang: + Made by me: tay to, giao diện sẽ khá xấu.. + Đi tìm 1 khung template mẫu: tiết kiệm thời gian, và giao diện sẽ khá đẹp và responsive *Giao diện Backend: hướng tới admin của hệ thống SẼ chọn 1 template admin có sẵn, backend ko chú trọng quá vào giao diện do chỉ hướng tới số lượng khá ít người dùng Gợi ý: 1 template rất thông dụng với admin là AdminLTE
